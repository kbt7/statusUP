# statusUP

ランキング画面から貼り付けたテキストを取り込み、時点間の **戦闘力の変動・順位の遷移・戦力上昇値ランキング** を表示するツール。

`index.html` をダブルクリックして開くだけ。インストールもビルドも不要。

## 使い方

1. `index.html` を開く。
2. ランキング画面をコピーして、上の枠に貼り付け（Ctrl+V）。
3. 日時を確認して「取り込む」。（既定は今の時刻。過去の分を入れるときだけ直す）
4. 時間をおいて 2 回目を取り込むと、変動が表示される。

貼り付けたテキストは **順位 → 名前 → 戦闘力** の順に並んでいることだけを頼りに読む。

- 上位 3 件が `2位 / 1位 / 3位` の順に並ぶレイアウトでもよい
- 「フレーム」「▼」などの余計な行、空行、タブ区切りは自動で読み飛ばす
- 戦闘力のカンマは有無どちらでもよい

## 見られるもの

- **カード** … 期間、合計上昇、人数（新規 / 圏外）、順位の上昇・下降人数
- **戦力上昇値ランキング** … 上昇値の大きい順。時間あたりの上昇も出る
- **上昇率ランキング** … 伸び率の大きい順
- **ランキング遷移** … 順位が動いた人だけ
- **全プレイヤー** … 今の順位順

比較する 2 時点は上のプルダウンで自由に選べる（既定は直近 2 件）。列見出しをクリックすると並べ替え、右上の枠で名前の絞り込み。

### 個人の推移

一覧の行をクリックすると、その人の詳細が開く。

- **サマリー** … 現在の順位（と最高順位）、現在の戦闘力、全期間の合計上昇、1 時間あたりの平均上昇
- **戦闘力の推移** … 折れ線グラフ。横軸は実際の時刻の間隔どおりに並ぶ
- **順位の推移** … 上に行くほど上位。その人の順位帯を拡大して表示するので、95 位 → 93 位のような小さな動きも見える
- **履歴表** … 各時点の順位・戦闘力・前回からの変動

グラフにマウスを乗せると、その時点の日時と数値が出る。圏外だった期間は線が途切れる。

プレイヤーは **名前で突き合わせる**。改名すると「圏外」＋「NEW」として扱われる。

### 圏外の扱い

ランキングに載っていないプレイヤーは **「圏外」** と表示される。圏外の人は**戦闘力そのものが取得できない**ので、分かるのは「その時点の最下位より下」ということだけ。

- 戦闘力の欄は、推測せずに **`115,995 未満`**（その時点の最下位の値）と表示する
- 上昇値・上昇率・時間あたりは計算できないため **`—`**（マウスを乗せると理由が出る）
- 順位変動は `5 → 圏外` / `圏外 → 31` のように、どちらへ動いたかを出す
- グラフでは圏外の時点に **×** を置き、線をつなげない

個人の詳細でも、いま圏外なら「現在の順位」は圏外と表示し、戦闘力は
**「最後に見えた戦闘力」** として最後に確認できた時点を明記する（古い値を現在の値として見せない）。

> 100 位までしか貼り付けていない場合、101 位以下の人も同じく「圏外」になる。表示される「◯◯未満」は、
> 貼り付けたランキングの最下位が基準。

## データの保存

取り込んだデータは **開いているブラウザの中だけに保存される**（localStorage）。閉じても次に開けばそのまま残る。サーバーへの送信は一切ない。

ブラウザの履歴・サイトデータを消すと一緒に消えるため、大事なら右上の「データの受け渡し」からバックアップを取っておく。

不要になったデータは、下の「保存済みデータ」から 1 件ずつ削除できる。

## 他の PC・スマホで見る（データの受け渡し）

localStorage は**ブラウザごと・サイトごとに分かれている**ので、PC で取り込んだデータはそのままではスマホに現れない。右上の **「データの受け渡し」** でファイルを渡す。

**渡す側**（データのある端末）は「書き出す」の **ダウンロード** でファイルを保存し、クラウドドライブや USB で渡したい端末へ移す。

**受け取る側**は「読み込む」の枠にファイルをドラッグ＆ドロップするか、枠をクリックして選ぶ。

読み込む前に**内容（件数・期間・追加/上書きの内訳）が表示され、確認してから取り込む**。今あるデータは消えず、**同じ日時のものだけ上書き、違う日時は追加**される。両方の端末で別々に取り込んでいても、合流させれば両方が残る。

statusUP が書き出したものでないファイルや壊れたファイルは、理由を出して弾く。手元のデータはそのまま保たれる。

> スマホからも取り込む場合、同じ日時のデータを両方の端末で作ると、後から読み込んだ側が勝つ。日時が違えば単純に増えていく。

## 公開するときの注意

`index.html` は **外部リソースを一切読み込まず、通信も行わない**。ページを配るだけで動く。

そのかわり、**ランキングのデータを `index.html` に書き込んではいけない**。埋め込むとページと一緒に全員へ配られてしまう。データは各自のブラウザの localStorage に入るのが正しい状態。

手元のランキングが入ったファイル（`my-data.json`、`data/`、`raw/`）は `.gitignore` で除外済み。公開時にアップロードするのは `index.html`（と `README.md`）だけでよい。
